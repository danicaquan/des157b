Project Feasibility Report

Features + Technologies

  • Record Submission + Addition (onto the page) –-- Back4App and Javascript
  • Navigation/Webpage –-- Javascript, HTML, CSS
    • drag around
    • zoom in/zoom out
    • responsive
    • click to read brick
    • submit form
  • Art (Bricks, Seal) –-- Procreate, Illustrator

Assessment

Largely, most of what I plan on implementing can be done via coding in Visual Studio Code. Any graphic images and art can be done in Procreate or Illustrator.

In terms of ability, I think it’s doable, at least the basic function i have planned. If I am stuck, I will do my research online and try to figure out how to make it work (or ask for help in class). One thing I am a little bit unsure of is how to make the submission stack next to each other in a random manner around the central Seal. I want to make it structured in a way similar to how it is in real life rather that posts/articles on a website. If I am unable to achieve this with my abilities, then I will still use the same page navigation method but the bricks will be arranged only horizontally. I likely will use a library like anime.js for some microinteractions.

I would love to add some customizable options like the ability to choose a profile picture and add some decorations to the bricks, or even the ability to upload photos and comment. However, this is largely based on if I have the time to do so after I finish the main function and design of the website. We will have to see. Ultimately, how I approach this will evolve as I get into the actual construction of the project and how I deal with the problems I will encounter (like if a library will make achieving a certain effect more simple, then I will implement that).
